# 04. USER_FLOW — SI-PINTAR

## 1. Ringkasan Dokumen

Dokumen ini menjelaskan alur penggunaan sistem **SI-PINTAR** berdasarkan role pengguna, proses utama sistem, status flow, dan error flow.

Dokumen ini digunakan sebagai acuan untuk:

- Memahami alur sistem dari awal sampai akhir.
- Menentukan urutan proses setiap role.
- Menentukan kondisi akses setiap fitur.
- Menentukan response sistem saat terjadi error.
- Membantu proses implementasi sistem oleh developer atau agent AI.

------

## 2. Aktor Sistem

## 2.1 Admin

Admin adalah pengguna yang mengelola seluruh proses training internal perusahaan.

Admin dapat:

- Login ke dashboard admin.
- Mengelola master data.
- Mengelola training.
- Mengelola materi.
- Mengelola pre-test dan post-test.
- Mengelola peserta training.
- Melakukan penilaian essay.
- Monitoring progress training.
- Melihat dan export laporan.
- Mengubah profil dan password.

## 2.2 Karyawan

Karyawan adalah pengguna yang mengikuti training yang ditugaskan.

Karyawan dapat:

- Login ke dashboard karyawan.
- Melihat training yang ditugaskan.
- Mengerjakan pre-test.
- Membuka dan mengunduh materi.
- Mengerjakan post-test.
- Melihat status penilaian.
- Melihat hasil training.
- Melihat riwayat training.
- Mengubah profil dan password.

## 2.3 Sistem

Sistem bertugas menjalankan validasi, hak akses, perubahan status, perhitungan nilai, dan response otomatis.

Sistem dapat:

- Memvalidasi login.
- Mengecek role user.
- Mengecek status akun aktif/nonaktif.
- Mengunci dan membuka akses training.
- Menghitung nilai pilihan ganda otomatis.
- Menandai essay sebagai menunggu penilaian.
- Mengupdate status progress training.
- Menampilkan toast, alert, modal, dan error validasi.

------

# 3. Alur Sistem Utama

## 3.1 Overview Alur Utama

```text
User membuka sistem
→ User login
→ Sistem validasi akun
→ Sistem cek role
→ User masuk dashboard sesuai role
→ Admin menyiapkan data dan training
→ Admin publish training
→ Karyawan mengikuti training
→ Karyawan mengerjakan pre-test
→ Karyawan membuka materi
→ Karyawan mengerjakan post-test
→ Sistem menghitung nilai pilihan ganda
→ Admin menilai essay jika ada
→ Sistem menentukan status kelulusan
→ Karyawan melihat hasil
→ Admin melihat monitoring dan laporan
→ Admin export PDF/Excel jika diperlukan
```

------

## 3.2 Flow Login

```text
User membuka halaman login
→ User mengisi username dan password
→ User klik Login
→ Sistem validasi username dan password
→ Sistem cek status akun
→ Sistem cek role
→ Sistem redirect ke dashboard sesuai role
```

## Output Login Berdasarkan Role

| Role     | Redirect           |
| -------- | ------------------ |
| Admin    | Dashboard Admin    |
| Karyawan | Dashboard Karyawan |

## Error Login

| Kondisi                 | Response Sistem                                          |
| ----------------------- | -------------------------------------------------------- |
| Username/password salah | Sistem menampilkan pesan error login                     |
| Akun nonaktif           | Sistem menolak login dan menampilkan pesan akun nonaktif |
| Role tidak valid        | Sistem menolak akses                                     |
| Session expired         | Sistem redirect ke halaman login                         |

Master data terdiri dari:

- Karyawan.
- Departemen.
- Jabatan.

```text
Admin membuka menu master data
→ Admin klik Tambah
→ Sistem menampilkan form tambah data
→ Admin mengisi data
→ Admin klik Simpan
→ Sistem validasi input
→ Sistem menyimpan data
→ Sistem menampilkan toast sukses

```

## Flow Edit Data

```text
Admin membuka data
→ Admin klik Edit
→ Sistem menampilkan form edit
→ Data lama otomatis terisi
→ Admin mengubah data
→ Admin klik Simpan Perubahan
→ Sistem validasi input
→ Sistem menyimpan perubahan

```

## Flow Nonaktifkan Data

```text
Admin klik Nonaktifkan
→ Sistem menampilkan custom modal konfirmasi
→ Admin klik Konfirmasi
→ Sistem mengubah status data menjadi Nonaktif
→ Sistem menampilkan toast sukses
```

## Flow Hapus Data

```text
Admin klik Hapus
→ Sistem menampilkan custom modal konfirmasi peringatan
→ Admin membaca dampak penghapusan
→ Admin klik Hapus Permanen
→ Sistem menghapus data
→ Sistem menampilkan toast sukses
```

## Catatan

- Data boleh dinonaktifkan.
- Data boleh dihapus permanen.
- Aksi hapus wajib menggunakan custom modal konfirmasi.
- Jika data masih terhubung dengan data lain, sistem harus menampilkan peringatan sebelum penghapusan.

## 4.1.8 Flow Mengelola Pre-Test

```text

→ Admin masuk tab Pre-Test
```

Catatan:

- Pre-test digunakan untuk mengukur kemampuan awal.
- Pre-test tidak menentukan kelulusan.
- Soal dapat berupa pilihan ganda dan essay.
- Urutan soal tidak random.
- Opsi jawaban tidak random.

## 4.1.9 Flow Mengelola Post-Test

```text
Admin membuka Detail Training

→ Admin mengatur passing grade

→ Admin menambahkan soal
→ Admin klik Simpan
→ Sistem menyimpan post-test
```

Catatan:

- Post-test menentukan kelulusan.
- Pengulangan hanya berlaku untuk post-test.

------

## 4.1.10 Flow Mengelola Peserta

```text
Admin membuka Detail Training
→ Admin masuk tab Peserta
→ Admin memilih jenis penugasan
→ Sistem menampilkan daftar target peserta
→ Admin menyimpan peserta
→ Sistem menugaskan training kepada peserta
```

Pilihan penugasan:

- Semua karyawan.
- Berdasarkan departemen.
- Berdasarkan jabatan.
- Pilih karyawan tertentu.

Catatan:

- Karyawan hanya bisa melihat training yang ditugaskan.
- Training tidak akan muncul pada karyawan yang tidak ditugaskan.
- Jika karyawan mengakses URL training yang bukan miliknya, sistem menolak akses.

------

## 4.1.11 Flow Publish Training

```text
Admin membuka Detail Training
→ Admin klik Publish
→ Sistem mengecek kelengkapan training
→ Jika ada data belum lengkap, sistem menampilkan warning
→ Admin tetap dapat melanjutkan publish jika diperlukan
→ Sistem menampilkan custom modal konfirmasi
→ Admin klik Publish
→ Status training berubah menjadi Published
→ Training dapat diakses oleh peserta yang ditugaskan
```

Warning yang dapat muncul:

- Training belum memiliki peserta.
- Training belum memiliki materi.
- Pre-test belum tersedia.
- Post-test belum tersedia.
- Tanggal training belum valid.

------

## 4.1.12 Status Training Flow

```text
Draft
→ Published
→ Closed
→ Archived
```

| Status    | Arti                     | Dampak                               |
| --------- | ------------------------ | ------------------------------------ |
| Draft     | Training masih disiapkan | Belum bisa diakses karyawan          |
| Published | Training sudah aktif     | Bisa diakses peserta yang ditugaskan |
| Closed    | Training sudah ditutup   | Tidak bisa dikerjakan lagi           |
| Archived  | Training diarsipkan      | Hanya untuk data arsip               |

------

## 4.1.13 Flow Penilaian Essay

```text
Admin membuka menu Penilaian
→ Sistem menampilkan daftar peserta yang menunggu penilaian
→ Admin klik Detail
→ Sistem menampilkan jawaban peserta
→ Pilihan ganda tampil read-only
→ Essay tampil untuk dinilai
→ Admin mengisi nilai essay
→ Admin klik Simpan Nilai
→ Sistem validasi nilai
→ Sistem menghitung nilai akhir
→ Sistem menentukan status kelulusan
→ Sistem menampilkan toast sukses
```

Catatan:

- Pilihan ganda dinilai otomatis oleh sistem.
- Essay dinilai manual oleh admin.
- Jika essay belum dinilai, status tetap Menunggu Penilaian.
- Setelah semua essay dinilai, sistem menentukan hasil akhir.

Filter monitoring:

## 4.2.5 Flow Training Dengan Test

```text
Karyawan membuka training
→ Sistem menampilkan tombol Kerjakan Pre-Test
→ Karyawan mengerjakan pre-test
→ Sistem menyimpan jawaban pre-test
→ Sistem menandai pre-test selesai
→ Materi terbuka
→ Karyawan membuka semua materi wajib
→ Sistem menandai materi sudah diakses
→ Post-test terbuka
→ Karyawan mengerjakan post-test
→ Sistem menyimpan jawaban post-test
→ Sistem menghitung nilai pilihan ganda
→ Jika tidak ada essay, sistem menampilkan hasil
→ Jika ada essay, status menjadi Menunggu Penilaian
→ Admin menilai essay
→ Sistem menampilkan hasil akhir
```

------

## 4.2.6 Flow Training Tanpa Test

```text
Karyawan membuka training
→ Sistem cek bahwa training tidak menggunakan test
→ Materi langsung terbuka
→ Karyawan membuka materi
→ Sistem mencatat materi sudah diakses
→ Jika semua materi wajib sudah dibuka, training dianggap selesai
→ Karyawan dapat melihat status selesai
```

Catatan:

- Flow tanpa test melewati pre-test dan post-test.
- Training selesai setelah materi wajib diakses.

------

## 4.2.7 Flow Membuka Materi

```text
Karyawan membuka tab Materi
→ Sistem menampilkan daftar materi aktif
→ Karyawan membuka materi
→ Sistem mencatat riwayat akses materi
→ Jika materi berupa file, karyawan dapat preview/download
→ Jika materi berupa link, karyawan membuka link
→ Sistem menandai materi sudah diakses
```

Aturan materi:

- Semua materi aktif wajib dibuka minimal satu kali.
- Post-test terbuka setelah semua materi wajib sudah diakses.
- Riwayat akses materi otomatis tersimpan dan tidak hilang.

------

## 4.2.8 Flow Pengerjaan Test

```text
Karyawan klik Kerjakan Test
→ Sistem menampilkan instruksi test
→ Karyawan klik Mulai Test
→ Sistem memulai timer
→ Sistem menampilkan soal
→ Karyawan mengisi jawaban
→ Karyawan klik Simpan Jawaban atau lanjut soal
→ Sistem menyimpan jawaban sementara
→ Karyawan klik Submit Test
→ Sistem menampilkan custom modal konfirmasi
→ Karyawan klik Submit
→ Sistem menyimpan dan mengunci jawaban
```

Catatan:

- Jawaban tersimpan saat user menyimpan atau berpindah soal.
- Jika karyawan keluar halaman, jawaban yang sudah tersimpan tidak hilang.
- Karyawan dapat melanjutkan test selama waktu masih tersisa.
- Setelah submit, jawaban tidak dapat diubah.

------

## 4.2.9 Flow Timer Test Habis

```text
Timer test berjalan
→ Waktu habis
→ Sistem otomatis submit jawaban yang sudah tersimpan
→ Sistem mengunci jawaban
→ Sistem menghitung nilai pilihan ganda
→ Sistem menentukan langkah berikutnya
```

Jika test memiliki essay:

```text
Jawaban dikirim
→ Status menjadi Menunggu Penilaian
→ Karyawan menunggu admin menilai essay
```

Jika test hanya pilihan ganda:

```text
Jawaban dikirim
→ Sistem menghitung nilai otomatis
→ Sistem menampilkan hasil
```

------

## 4.2.10 Flow Setelah Pre-Test

```text
Karyawan submit pre-test
→ Sistem menyimpan hasil pre-test
→ Sistem menandai pre-test selesai
→ Sistem tidak menampilkan jawaban benar

```

Catatan:

- Pre-test tidak menentukan kelulusan.
- Karyawan cukup melihat status selesai.
- Jawaban benar dan pembahasan tidak ditampilkan.

------

## 4.2.11 Flow Setelah Post-Test

## Jika Semua Soal Pilihan Ganda

```text
Karyawan submit post-test
→ Sistem menghitung nilai otomatis
→ Sistem membandingkan nilai dengan passing grade
→ Sistem menentukan status Lulus atau Tidak Lulus
→ Karyawan dapat melihat hasil
```

## Jika Ada Essay

```text
Karyawan submit post-test
→ Sistem menghitung nilai pilihan ganda
→ Sistem menandai essay perlu dinilai admin
→ Status menjadi Menunggu Penilaian
→ Karyawan melihat status Menunggu Penilaian
→ Admin menilai essay
→ Sistem menghitung nilai akhir
→ Sistem menentukan status Lulus atau Tidak Lulus
→ Karyawan dapat melihat hasil
```

------

## 4.2.12 Flow Tidak Lulus dan Mengulang

```text
Sistem menentukan karyawan tidak lulus
→ Sistem cek batas percobaan post-test
→ Jika masih ada percobaan
→ Sistem menampilkan tombol Ulangi Test
→ Karyawan klik Ulangi Test
→ Sistem membuka post-test ulang
→ Karyawan mengerjakan post-test ulang
→ Sistem menyimpan attempt baru
→ Sistem menentukan hasil akhir terbaru
```

Catatan:

- Pengulangan hanya berlaku untuk post-test.
- Pre-test tidak diulang.
- Materi tidak perlu diulang dari awal.
- Setiap percobaan post-test disimpan sebagai riwayat.

------

## 4.2.13 Flow Percobaan Habis

```text
Karyawan tidak lulus
→ Sistem cek batas percobaan
→ Percobaan sudah habis
→ Sistem tidak menampilkan tombol Ulangi Test
→ Status akhir tetap Tidak Lulus
→ Karyawan hanya dapat melihat hasil
```

------

# 5. State / Status Flow

## 5.1 Status Progress Karyawan

```text
Belum Mulai
→ Sedang Berjalan
→ Menunggu Penilaian
→ Lulus / Tidak Lulus
→ Mengulang
```

## Detail Transisi Status

| Dari Status        | Ke Status          | Kondisi                                                      |
| ------------------ | ------------------ | ------------------------------------------------------------ |
| Belum Mulai        | Sedang Berjalan    | Karyawan mulai membuka training/pre-test                     |
| Sedang Berjalan    | Menunggu Penilaian | Post-test memiliki essay yang belum dinilai                  |
| Sedang Berjalan    | Lulus              | Nilai post-test memenuhi passing grade                       |
| Sedang Berjalan    | Tidak Lulus        | Nilai post-test di bawah passing grade                       |
| Tidak Lulus        | Mengulang          | Karyawan masih punya sisa percobaan                          |
| Mengulang          | Sedang Berjalan    | Karyawan mulai post-test ulang                               |
| Menunggu Penilaian | Lulus              | Essay selesai dinilai dan nilai memenuhi passing grade       |
| Menunggu Penilaian | Tidak Lulus        | Essay selesai dinilai dan nilai tidak memenuhi passing grade |

------

## 5.2 Status Penilaian

```text
Menunggu Penilaian
→ Selesai Dinilai
```

| Status             | Arti                                     |
| ------------------ | ---------------------------------------- |
| Menunggu Penilaian | Masih ada essay yang belum dinilai admin |
| Selesai Dinilai    | Semua essay sudah dinilai                |

------

## 5.3 Status Data Master

```text
Aktif
→ Nonaktif
```

| Status   | Dampak                                 |
| -------- | -------------------------------------- |
| Aktif    | Data dapat digunakan                   |
| Nonaktif | Data tidak digunakan untuk proses baru |

# 6. Error Flow

## 6.1 Authentication Error

| Kondisi         | Trigger                         | Response Sistem   | Pesan UI                                        |
| --------------- | ------------------------------- | ----------------- | ----------------------------------------------- |
| Login gagal     | Username/password salah         | Login ditolak     | Username atau password salah                    |
| Akun nonaktif   | User login dengan akun nonaktif | Login ditolak     | Akun Anda nonaktif                              |
| Session expired | Session habis                   | Redirect ke login | Sesi Anda telah berakhir, silakan login kembali |

------

## 6.2 Authorization Error

| Kondisi                       | Trigger                                       | Response Sistem           | Pesan UI                                 |
| ----------------------------- | --------------------------------------------- | ------------------------- | ---------------------------------------- |

# 7. Catatan UX Flow

## 7.1 Satu Tombol Utama untuk Karyawan

Karyawan hanya diarahkan dengan satu tombol utama sesuai status terakhir.

Contoh:

```text
Belum mulai → Kerjakan Pre-Test
Pre-test selesai → Buka Materi
Materi diakses → Kerjakan Post-Test
Menunggu Penilaian → Menunggu Penilaian
Selesai → Lihat Hasil
Tidak lulus dan masih bisa mengulang → Ulangi Test
```

Tujuan:

- Karyawan tidak bingung.
- Flow training tetap terkontrol.
- Sistem menentukan langkah berikutnya secara otomatis.

------

## 7.2 Custom Modal untuk Aksi Penting

Custom modal digunakan untuk:

- Hapus data.
- Publish training.
- Tutup training.
- Submit test.
- Export data jika diperlukan.

Tidak boleh menggunakan:

```text
alert()
confirm()
prompt()
```

------

## 7.3 Toast untuk Feedback Sukses

Toast digunakan untuk feedback ringan.

Contoh:

```text
Data berhasil disimpan.
Training berhasil dipublish.
Nilai essay berhasil disimpan.
Laporan berhasil diexport.
```

------

## 7.4 Inline Validation untuk Error Form

Error form harus tampil di bawah field yang bermasalah.

Contoh:

```text
Nama karyawan wajib diisi.
Password baru dan konfirmasi password tidak sama.
Passing grade harus berupa angka 0 sampai 100.
```

------

## 7.5 Alert untuk Kondisi Penting

Alert digunakan untuk kondisi yang perlu diperhatikan user.

Contoh:

```text
Training masih Draft. Publish training agar dapat diakses peserta.
Post-test masih terkunci. Silakan akses materi terlebih dahulu.
Masih ada essay yang menunggu penilaian.
```

------
