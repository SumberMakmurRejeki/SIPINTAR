# Feature Specification — SI-PINTAR

## 1. Ringkasan

**SI-PINTAR** adalah sistem informasi pelatihan internal perusahaan untuk membantu admin mengelola training, materi, test, penilaian, progres karyawan, dan laporan.

Sistem dibuat dengan konsep:

- Mudah digunakan admin dan karyawan.
- Alur kerja sederhana.
- Tidak berbelit-belit.
- Karyawan cukup mengikuti tombol utama sesuai status.
- Pilihan ganda dinilai otomatis oleh sistem.
- Essay dinilai manual oleh admin.

------

## 2. Role Pengguna

| Role     | Deskripsi                                                    |
| -------- | ------------------------------------------------------------ |
| Admin    | Mengelola data master, training, materi, test, peserta, penilaian, monitoring, dan laporan. |
| Karyawan | Mengikuti training, mengerjakan pre-test, membuka materi, mengerjakan post-test, dan melihat hasil. |

------

## Tabel/Card

| Kolom          |
| -------------- |
| Judul Training |
| Progress       |
| Status         |

## Tombol Utama Berdasarkan Status

| Kondisi                              | Tombol             |
| ------------------------------------ | ------------------ |
| Belum mengerjakan pre-test           | Kerjakan Pre-Test  |
| Pre-test selesai                     | Buka Materi        |
| Materi sudah diakses                 | Kerjakan Post-Test |
| Ada essay belum dinilai              | Menunggu Penilaian |
| Training selesai                     | Lihat Hasil        |
| Tidak lulus dan masih bisa mengulang | Ulangi Test        |

## Alur Karyawan

```text
Pre-Test
→ Materi
→ Post-Test
→ Penilaian
→ Hasil
```

## Catatan Zero Thinking

Karyawan hanya melihat satu tombol utama sesuai status terakhir, sehingga tidak perlu memilih alur sendiri.

# 13. Aturan Penting Sistem

1. Karyawan hanya bisa mengakses training yang ditugaskan.
2. Alur karyawan dikunci otomatis oleh sistem.
3. Pre-test wajib dikerjakan sebelum materi terbuka.
4. Materi wajib diakses sebelum post-test terbuka.
5. Kelulusan ditentukan dari nilai post-test.
6. Nilai pre-test hanya sebagai pembanding kemampuan awal.
7. Pilihan ganda dinilai otomatis oleh sistem.
8. Essay dinilai manual oleh admin.
9. Jika essay belum dinilai, status menjadi menunggu penilaian.
10. Karyawan hanya melihat nilai dan status.
11. Karyawan tidak melihat jawaban benar.
12. Laporan hanya bisa diakses admin.
13. Export tersedia dalam PDF dan Excel.
14. Import tidak tersedia pada MVP.

# 14. Catatan Zero Thinking

## Untuk Admin

- Menu sidebar dibuat ringkas.
- Semua pengaturan training berada di satu tempat.
- Detail training memakai tab.
- Penilaian menampilkan data yang butuh tindakan admin.
- Monitoring Progress fokus untuk pantauan.
- Laporan Training fokus untuk filter dan export.

## Untuk Karyawan

- Karyawan hanya melihat training yang ditugaskan.
- Sistem menampilkan satu tombol utama sesuai status.
- Materi dan post-test terkunci otomatis.
- Karyawan tidak perlu memilih alur.
- Hasil akhir hanya menampilkan nilai dan status.
